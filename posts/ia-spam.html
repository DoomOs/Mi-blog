Introducci√≥n y contexto

El spam en los correos y mensajes es un problema que afecta a millones de usuarios diariamente. Estos mensajes pueden contener contenido no deseado, publicidad enga√±osa o incluso amenazas como phishing y malware.

Con el crecimiento exponencial de la comunicaci√≥n digital, identificar manualmente el spam es ineficiente. Aqu√≠ es donde entra la Inteligencia Artificial (IA): los modelos de Machine Learning pueden aprender patrones en los mensajes para clasificar autom√°ticamente si un mensaje es spam o no.

Preprocesamiento del texto

Antes de entrenar un modelo de Machine Learning, es necesario limpiar y normalizar el texto para eliminar ruido y mejorar la representaci√≥n de los datos.

Pasos del preprocesamiento:

  1. Conversi√≥n a min√∫sculas: Para evitar diferencias entre palabras como "Hola" y "hola".
  2. Eliminaci√≥n de caracteres especiales y n√∫meros: Se eliminan caracteres innecesarios.
  3. Eliminaci√≥n de stopwords: Palabras como "el", "la", "de" que no aportan significado.
  4. Stemming y lematizaci√≥n: Para reducir palabras a su forma base (por ejemplo, "corriendo" ‚Üí "correr").

Se puede utilizar spaCy o NLTK para realizar este preprocesamiento (m√°s informaci√≥n sobre spaCy).

Extracci√≥n de caracter√≠sticas

Para que un algoritmo de Machine Learning pueda interpretar texto, es necesario convertir las palabras en valores num√©ricos. Una t√©cnica com√∫n para esto es TF-IDF (Term Frequency - Inverse Document Frequency)**.

¬øQu√© es TF-IDF?

Es una t√©cnica que mide la importancia de una palabra en un conjunto de documentos. Penaliza palabras comunes y resalta las m√°s relevantes en el contexto.

Ejemplo: En mensajes de spam, palabras como "ganar", "premio" o "dinero" suelen tener un puntaje TF-IDF alto.

Otra t√©cnica que podemos incluir es el uso de N-gramas, que considera combinaciones de palabras en lugar de palabras individuales. Los bigrams y trigrams pueden mejorar la precisi√≥n del modelo al capturar m√°s contexto.

Selecci√≥n y entrenamiento del modelo

Para esta tarea, usaremos Regresi√≥n Log√≠stica, un modelo simple pero efectivo para clasificaci√≥n binaria.

¬øPor qu√© Regresi√≥n Log√≠stica?

  • Es f√°cil de interpretar.
  • Funciona bien en textos representados con TF-IDF.
  • Es r√°pido y eficiente en datasets peque√±os/medianos.

Usaremos Pipeline de scikit-learn para integrar el preprocesamiento y entrenamiento en un solo flujo.

pipeline = make_pipeline(
    TfidfVectorizer(max_features=1000, ngram_range=(1,2)),

Ejemplo de uso en una API Flask:

from flask import Flask, request, jsonify
import joblib

app = Flask(__name__)
model = joblib.load("spam_classifier.pkl")

@app.route("/predict", methods=["POST"])
def predict():
    text = request.json["text"]
    prediction = model.predict([text])
    return jsonify({"spam": bool(prediction[0])})
